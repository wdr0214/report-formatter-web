# 汇报材料格式整理工具 浏览器本地版

这是一个纯静态网页版本，可本地打开，也可部署到 GitHub Pages。文件只在本机浏览器内处理，不上传服务器。

## 使用方法

1. 推荐运行 `python start-local-server.py`，然后在浏览器打开提示的本地地址；也可以直接打开 `index.html`。
2. 页面默认使用同目录下的 `汇报材料模板.docx`；只有需要更换模板时才点击“更换模板”选择新的 `.docx` 模板。
3. 点击“选择文件”或“选择文件夹”，选择待处理文件；如果列表中有不想处理的文件，可点击该行“删除”。
4. 点击“调整格式”。
5. 点击“预览”查看处理结果。
6. 点击“下载”保存处理后的文件。

## 支持范围

- 稳定支持 `.docx`。
- `.doc` 和 `.wps` 会提示“请先另存为 docx”，不直接处理。
- 下载位置由浏览器控制。
- 默认模板文件需要和 `index.html` 放在同一目录；部署到 GitHub Pages 时也会随页面一起发布。
- 浏览器版会尽量打开修订跟踪设置，但不保证完整 Office 可视修订痕迹。

## 规则

- 最多识别 4 级标题：`一、`、`（一）`、`1.`、`（1）`。
- 正文加粗保留。
- 表格内不调整格式，只检查文字问题。
- 图表标题、自动目录、首页文头不改。
- 总标题下方 3 行内居中内容不调格式，只改英文括号。
- 四级标题不加粗。
- 三级标题点号后空格删除。
- 英文括号改中文括号，英文双引号成对改中文引号。

## GitHub Pages

将整个 `ReportFormatterWebLocal` 目录提交到 GitHub 仓库后，可用 GitHub Pages 部署。部署后仍然是浏览器本地处理，不会把文件上传到 GitHub。

## 注意

页面默认从 CDN 加载 JSZip。部署到 GitHub Pages 或联网使用时可以直接加载；如果要完全离线使用，需要把 JSZip 下载到本目录并把 `index.html` 中的脚本地址改成本地文件。
